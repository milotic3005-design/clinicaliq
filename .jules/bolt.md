## 2025-04-28 - Replace Async Crypto Hash with Synchronous String Lookup for In-Memory Cache
**Learning:** Using Web Crypto API (`crypto.subtle.digest`) for generating cache keys for an in-memory `Map` adds significant asynchronous overhead (benchmark showed 1368ms for 10k ops vs 1.5ms for string ops). A synchronous, deterministic string interpolation like `${query.toLowerCase()}:${queryType}` provides an equally unique and functionally identical map key without the need for async resolution or hashing.
**Action:** When storing state in simple in-memory structures like `Map`, avoid using cryptographic hashes for key generation unless specifically required for privacy/security reasons; standard string concatenation is measurably faster and enables fully synchronous code paths.
